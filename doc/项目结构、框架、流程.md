**设计原则：**

- **分层清晰：** `core` 是基础设施，`agents` 是业务逻辑，`graph` 是编排层，互不污染
- **单一职责：** 每个 agent 一个文件，方便测试和替换
- **渐进式：** `memory/`、`feedback/`、`tools/` 暂时空着，目录先建好，代码逐步填充



## 二、开发框架选型与版本锁定

| 层             | 技术选型                         | 版本                          | 选型理由                                                     |
| :------------- | :------------------------------- | :---------------------------- | :----------------------------------------------------------- |
| **Agent编排**  | LangGraph                        | ≥0.2.0                        | 原生支持状态图、Checkpoint、子图，比LangChain的AgentExecutor更灵活 |
| **LLM调用**    | LangChain + OpenAI SDK           | langchain≥0.3.0, openai≥1.0.0 | LangChain封装统一接口，底层用OpenAI SDK兼容所有厂商          |
| **向量数据库** | ChromaDB                         | ≥0.5.0                        | 轻量、Python原生、支持持久化，个人项目首选                   |
| **知识图谱**   | Neo4j                            | 社区版5.x                     | 后期温记忆需要，前期可暂不安装                               |
| **Web框架**    | FastAPI                          | ≥0.110.0                      | 异步、高性能，后续暴露API用                                  |
| **配置管理**   | python-dotenv + Pydantic         | pydantic≥2.0                  | dotenv读文件，Pydantic做配置校验                             |
| **文档解析**   | PyMuPDF + python-docx + markdown | 最新稳定版                    | 分别处理PDF、Word、Markdown                                  |
| **测试框架**   | pytest                           | ≥8.0                          | Python标准测试框架                                           |

## 三、开发流程与阶段规划

我们按 **6 个阶段** 渐进式开发，每阶段有明确的输入、输出和验收标准。

### 阶段全景图

text

```
Phase 1: 核心骨架（MVP）
    ↓
Phase 2: RAG 知识库接入
    ↓
Phase 3: 短期记忆 + Checkpoint 回退
    ↓
Phase 4: 长期记忆系统（三层架构）
    ↓
Phase 5: 反思机制 + 多Agent协作
    ↓
Phase 6: 反馈学习 + 生产保障
```



### 各阶段详细规划

#### Phase 1: 核心骨架（当前阶段）

**目标：** 跑通 “输入 → 意图理解 → 任务拆解 → 总结输出” 的最简流程

| 任务              | 文件                                                         | 验收标准                       |
| :---------------- | :----------------------------------------------------------- | :----------------------------- |
| 1.1 项目初始化    | `config/`, `.env.example`, `requirements.txt`                | 配置加载成功，LLM连通          |
| 1.2 定义状态结构  | `state/agent_state.py`                                       | Task和AgentState定义清晰       |
| 1.3 实现三个Agent | `agents/intent_agent.py`, `decompose_agent.py`, `summary_agent.py` | 各Agent单独测试通过            |
| 1.4 编排主图      | `graph/main_graph.py`                                        | 线性流程跑通，返回合理拆解结果 |
| 1.5 命令行入口    | `main.py`                                                    | 对话交互流畅                   |

**验收用例：** 输入 “我想做一个监控AI论文的工具”，输出包含 4-6 个有序步骤，步骤间有依赖关系。

------

#### Phase 2: RAG 知识库接入

**目标：** 让Agent在拆解任务时能参考你的私有AI知识库

| 任务           | 文件                     | 验收标准                              |
| :------------- | :----------------------- | :------------------------------------ |
| 2.1 文档加载器 | `rag/document_loader.py` | 能加载PDF/Word/Markdown，输出统一文本 |
| 2.2 智能分块器 | `rag/text_splitter.py`   | 按标题/段落分块，保留元数据           |
| 2.3 向量化入库 | `rag/retriever.py`       | 文档→向量→Chroma入库，可检索          |
| 2.4 集成到主图 | `graph/main_graph.py`    | 在拆解前先检索知识，注入到Prompt      |

**验收用例：** 放一份AI部署相关的PDF到 `data/knowledge_base/`，问“怎么部署模型”，Agent能引用PDF中的内容。

------

#### Phase 3: 短期记忆 + Checkpoint 回退

**目标：** 支持多轮对话，任务失败能回滚

| 任务               | 文件                  | 验收标准                       |
| :----------------- | :-------------------- | :----------------------------- |
| 3.1 对话历史管理   | `memory/` (初版)      | 多轮对话保持上下文连贯         |
| 3.2 Checkpoint机制 | `graph/main_graph.py` | 每个节点执行后自动保存状态快照 |
| 3.3 回退逻辑       | `graph/main_graph.py` | 模拟失败，能回退到上一个检查点 |

**验收用例：** 对话中说“第三步我不满意，换一种方式”，Agent能回到第三步重新生成。

------

#### Phase 4: 长期记忆系统

**目标：** 实现热-温-冷三层记忆架构

| 任务                | 文件                        | 验收标准              |
| :------------------ | :-------------------------- | :-------------------- |
| 4.1 温记忆-向量库   | `memory/vector_store.py`    | 偏好、事实持久化存储  |
| 4.2 温记忆-知识图谱 | `memory/knowledge_graph.py` | 三元组抽取、Neo4j写入 |
| 4.3 记忆管理器      | `memory/memory_manager.py`  | 记录和检索的统一入口  |
| 4.4 记忆优化器      | `memory/memory_manager.py`  | 去重、合并、过期清理  |

**验收用例：** 第一次对话说“我是Python用户”，下次对话问“推荐什么语言”，Agent回答偏好Python。

------

#### Phase 5: 反思机制 + 多Agent协作

**目标：** Agent能发现错误并自我修正

| 任务             | 文件                                 | 验收标准                     |
| :--------------- | :----------------------------------- | :--------------------------- |
| 5.1 反思Agent    | `agents/reflect_agent.py`            | 能检查输出质量，发现逻辑矛盾 |
| 5.2 局部重试     | `graph/main_graph.py`                | 格式错误时自动重试最多3次    |
| 5.3 辩论子图     | `graph/subgraphs/debate_subgraph.py` | 两个Agent矛盾时启动辩论      |
| 5.4 人工介入接口 | `graph/main_graph.py`                | 无法解决时挂起，等待人工输入 |

**验收用例：** 故意让拆解结果有循环依赖，Agent能检测并修正。

------

#### Phase 6: 反馈学习 + 生产保障

**目标：** Bad Case闭环、灰度发布、熔断机制

| 任务             | 文件                          | 验收标准                       |
| :--------------- | :---------------------------- | :----------------------------- |
| 6.1 Bad Case捕获 | `feedback/badcase_catcher.py` | 差评、报错、重复提问等自动标记 |
| 6.2 金标测试集   | `feedback/gold_dataset.py`    | 修复的Case强制加入回归测试     |
| 6.3 评测流水线   | `feedback/evaluator.py`       | 发版前自动跑金标集             |
| 6.4 API化        | `api/`                        | FastAPI暴露接口                |
| 6.5 熔断中间件   | `api/middleware.py`           | 错误率超标自动回滚             |
